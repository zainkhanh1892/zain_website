01 — Problem

SOC Analysts Are Drowning in Alerts

Security Operations Centers are facing an alert overload crisis. Analysts at Crest Data receive hundreds of malsite alerts daily, each requiring them to manually copy domains between systems, query threat intelligence APIs, write documentation, and update tickets — the same mechanical process, hundreds of times per shift.

"I spend a good chunk of my day triaging malsite alerts. By the time I finish, there's no time left for actual threat hunting." — Crest Data SOC Analyst

The real cost isn't just wasted time — it's sophisticated threats slipping through while analysts are buried in routine work. Decision quality also varies by analyst and by shift, creating inconsistency the team can't audit or fix.

02 — Goals

What We Set Out to Build

Our MVP target was a fully automated triage pipeline that handles the end-to-end alert workflow — from detecting a new Jira ticket to posting a structured verdict — without any analyst intervention for clear-cut cases.

We also required the system to be explainable (analysts see the reasoning behind every verdict), modular (swap any component without rebuilding), and trustworthy (confidence scores guide human judgment rather than replace it).

Key ConceptWhy It Matters
MCP Server Architecture Open standard protocol connects Jira, VirusTotal, and the LLM without hardcoding integrations — swap any component without rebuilding the system.
LLM Classification (GPT-4o) AI understands context like an analyst instead of following rigid rules, catching edge cases that pattern matching misses.
VirusTotal Enrichment Detection ratio, reputation score, and threat categories give the LLM structured signal — replacing the manual lookup step entirely.
Explainable Verdicts + Fallback Confidence scores and natural-language reasoning build analyst trust. Rule-based sanity checks keep things running when the LLM hits edge cases.
03 — Process

How We Built It — Team Perspective

As a team, we followed a research-first approach — interviewing SOC analysts at Crest Data, mapping the existing workflow step by step, and validating assumptions before writing a line of code. A key pivot came from stakeholder feedback: our initial Jira output format fell flat with analysts, so we restructured it to verdict-first, then recommended actions, then supporting evidence.

We validated continuously against real Crest Data malsite alerts — enriching 15+ live indicators, comparing AI verdicts to known analyst decisions, and achieving over 85% accuracy. Reputation weighting and a data freshness flag were added per analyst feedback mid-build.

05 — Takeaways

What I've Learned So Far

  • 1 Prompt design is real engineering. Writing prompts that produce consistent, structured, edge-case-safe outputs from GPT-4o took far more iteration than expected. Small wording changes had large effects on classification accuracy.
  • 2 MCP is a genuinely powerful integration pattern. Modular servers communicating over a standard protocol meant we could swap VirusTotal for another threat intel source — or Jira for ServiceNow — without touching the core pipeline.
  • 3 Stakeholder feedback changes everything. Our first output format felt complete to us but fell flat with analysts. Restructuring to verdict-first came directly from watching users scan the output — a reminder that user research doesn't stop after discovery.
  • 4 Ethical AI in enterprise contexts requires upfront design decisions. Confidence scores, fallback rules, and audit trails weren't afterthoughts — they were what made the system safe enough for a real SOC to actually use.
  • 5 Real client data is different from toy datasets. Live Jira tickets and VirusTotal results from actual malsite alerts introduced edge cases no academic dataset would have prepared me for.
